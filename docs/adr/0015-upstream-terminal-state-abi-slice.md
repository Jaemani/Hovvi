# ADR 0015: Upstream Terminal State ABI Slice

## Status

Accepted

## Context

ADR 0014 added an upstream-backed C ABI packet slice. It could create upstream
crypto/session state and decrypt inbound mosh packets, but it did not yet use
upstream terminal or user-input state.

The next roadmap step is to make the ABI more like the mobile engine boundary:
inbound server packets should become terminal renderer bytes, and user
input/resize should become outbound encrypted mosh packets.

## Decision

Extend the repository-only upstream ABI implementation with upstream terminal
and user state:

- `hovvi_mosh_core_receive_packet` decrypts a server-directed packet, validates
  its host-diff protobuf before calling upstream `Terminal::Complete`, applies
  the host diff, and returns bytes generated by upstream `Terminal::Display`.
- `hovvi_mosh_core_send_user_input` converts raw mobile input bytes into
  upstream `Network::UserStream`, serializes the user diff, wraps it in
  `Network::Packet`, encrypts it, and returns one outbound packet frame.
- `hovvi_mosh_core_resize` serializes an upstream resize event through the same
  outbound packet path.
- malformed host diffs return `HOVVI_MOSH_PROTOCOL_ERROR` at the ABI boundary
  instead of allowing upstream fatal assertions to abort the process.

The shipped MIT scaffold remains unchanged. The upstream terminal/state slice is
compiled only by `native:upstream-check`.

## Build Notes

The upstream terminal slice requires generated `userinput.pb.*` and
`hostinput.pb.*` in addition to `transportinstruction.pb.*`. It also compiles
the vendored terminal and statesync source groups and links against curses for
`Terminal::Display` capability initialization.

The Hovvi-owned Apple config shim defines `HAVE_CURSES_H` for this repository
validation build. Vendored upstream files are not modified.

## Consequences

- The C ABI now has a real upstream-backed terminal output path for valid server
  host diffs.
- The C ABI now has real upstream-backed outbound packet paths for user input
  and resize events.
- Full mosh transport behavior is still incomplete: retransmission, ack,
  prediction timing, shutdown, and real relay/mosh-server harness coverage remain
  future milestones.
